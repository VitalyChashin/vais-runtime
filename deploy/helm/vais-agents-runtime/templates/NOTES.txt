1. Wait for the pod(s) to reach Ready:

     kubectl rollout status deployment/{{ include "vais-agents-runtime.fullname" . }} -n {{ .Release.Namespace }}

2. Port-forward the HTTP control plane:

     kubectl port-forward svc/{{ include "vais-agents-runtime.fullname" . }} 8080:{{ .Values.service.port }} -n {{ .Release.Namespace }}

3. Verify the v0.11 OpenAPI spec is served:

     curl http://localhost:8080/openapi/v1.json | head

4. Install the vais CLI and point it at the service:

     dotnet tool install -g Vais.Agents.Cli
     vais config set base-url http://localhost:8080

5. Apply a sample Agent manifest. **v0.16 limitation:** invoke returns
   501 urn:vais-agents:agent-not-instantiable — manifest-driven agent
   instantiation is Pillar B (v0.17). Create / Get / Delete already work
   against the registry.

     vais apply -f agent.yaml
     vais get agents
