# Guide: wire a custom tool

5-minute recipe. Defines a tool, registers it, drives one turn through the agent, observes the dispatch.

## 1. Write the tool

Option A — hand-rolled `ITool` (full control over schema, description, error handling):

```csharp
using System.Text.Json;
using Vais.Agents;

sealed class LookupOrderTool(IOrderStore orders) : ITool
{
    public string Name => "lookup_order";
    public string Description => "Fetch the shipping status of an order by its id.";
    public JsonElement ParametersSchema { get; } = JsonDocument.Parse(
        """
        {
          "type": "object",
          "properties": {
            "orderId": { "type": "string", "description": "The order id, e.g. 'ORD-12345'." }
          },
          "required": ["orderId"]
        }
        """).RootElement;

    public async Task<string> InvokeAsync(JsonElement args, CancellationToken ct = default)
    {
        var orderId = args.GetProperty("orderId").GetString()!;
        var order = await orders.GetAsync(orderId, ct);
        return order is null
            ? $"Order {orderId} not found."
            : $"Order {orderId}: status {order.Status}, ETA {order.Eta:yyyy-MM-dd}.";
    }
}
```

Option B — typed shortcut via `Tool.FromFunc` (schema generated by STJ):

```csharp
using Vais.Agents.Core;

sealed record LookupInput(string OrderId);

var tool = Tool.FromFunc<LookupInput, string>(
    name: "lookup_order",
    description: "Fetch the shipping status of an order by its id.",
    handler: async (input, ct) =>
    {
        var order = await orders.GetAsync(input.OrderId, ct);
        return order is null
            ? $"Order {input.OrderId} not found."
            : $"Order {input.OrderId}: status {order.Status}, ETA {order.Eta:yyyy-MM-dd}.";
    });
```

Hand-rolled when you need custom description-per-parameter or strict-mode schema dialects; typed shortcut otherwise.

## 2. Register

Minimal in-memory registry:

```csharp
sealed class InMemoryToolRegistry(params ITool[] tools) : IToolRegistry
{
    public IReadOnlyList<ITool> Tools { get; } = tools;
    public ITool? GetByName(string name) => Tools.FirstOrDefault(t => t.Name == name);
}

var registry = new InMemoryToolRegistry(tool /* from step 1 */);
```

For discovered tools (MCP, plugin catalogues), use `AggregatingToolRegistry.BuildAsync(staticTools, sources)`.

## 3. Wire into the agent

```csharp
var agent = new StatefulAiAgent(
    provider,
    new StatefulAgentOptions
    {
        ToolRegistry = registry,
        SystemPrompt = "Help customers. Use lookup_order to check order status.",
    });

var reply = await agent.AskAsync("Where's my order ORD-12345?");
Console.WriteLine(reply);
```

The model emits a tool call, the outer loop dispatches via `IToolCallDispatcher`, the result feeds back to the model, it generates a final answer.

## 4. Observe the dispatch

Add an event bus to see what happened:

```csharp
using Vais.Agents.Hosting.InMemory;

var bus = new InMemoryAgentEventBus();
using var sub = bus.Subscribe((@event, ct) =>
{
    Console.WriteLine($"  [{@event.At:HH:mm:ss.fff}] {@event.GetType().Name}");
    return ValueTask.CompletedTask;
});

var agent = new StatefulAiAgent(
    provider,
    new StatefulAgentOptions
    {
        ToolRegistry = registry,
        EventBus = bus,
    });
```

Running a tool-using turn prints:

```
  [HH:mm:ss.123] TurnStarted
  [HH:mm:ss.456] ToolCallStarted
  [HH:mm:ss.512] ToolCallCompleted
  [HH:mm:ss.789] TurnCompleted
```

## 5. Guard it

A tool that deletes things needs approval. Add an `IToolGuardrail`:

```csharp
sealed class DestructiveToolApproval : IToolGuardrail
{
    public Task<GuardrailOutcome> BeforeInvokeAsync(ToolCallRequest call, AgentContext ctx, CancellationToken ct)
        => call.ToolName.StartsWith("delete_", StringComparison.OrdinalIgnoreCase)
            ? Task.FromResult(GuardrailOutcome.Interrupt(
                new AgentInterrupt(Guid.NewGuid().ToString("N"),
                    $"Approval required to run '{call.ToolName}'.", call.Arguments),
                "destructive-tool-approval"))
            : Task.FromResult(GuardrailOutcome.Pass);
}

var agent = new StatefulAiAgent(
    provider,
    new StatefulAgentOptions
    {
        ToolRegistry = registry,
        ToolGuardrails = new[] { new DestructiveToolApproval() },
    });
```

See [add input/output guardrails](add-input-output-guardrails.md) for the other two layers.

## 6. Things that catch people

- **Name regex.** `ITool.Name` must match `[A-Za-z0-9_-]+`. Dots, spaces, and most Unicode don't fly. Adapters validate on registration.
- **`JsonElement` is cloneable but not owning.** If you store incoming `arguments` past the `InvokeAsync` call, `.Clone()` them first.
- **Return `string`, not objects.** The tool result feeds back to the model as text. If you want to return structured data, serialise to JSON yourself — the model will see that as a string and parse as it sees fit.
- **Cancellation.** Honour the `CancellationToken`. The agent cancels tool calls when the outer run is cancelled.
